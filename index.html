YaaT: Yet Another Annotation Tool

YAAT is an online software annotation tool that will allow the user to tag entities in the image, since the user is a precious resource for this project the software is designed like a classical web application. In fact this architecture allows the software to handle multiple sessions, this architecture is obviously cross OS.

YAAT is divided in 3 layers:

  1. Front End
  2. Back End
  3. DBMS
This structure of YAAT allows the System to be highly maintainable and allows the development to be concurrent, since using this structure we can define API that each module offers to other modules. The Front End will take care of the user input using HTML and JavaScript, building an interactive web page that allow the user to browse images, to annotate images, and to semantically search for images. All this functionalities use the APIs offered by the BackEnd. This layer is going to use web technologies like JavaScript, HTML, and CSS. The BackEnd is going to be written in python since this is a great prototyping language and through the framework FLASK we can easily implement RESTful web services. For the communication between FrontEnd and BackEnd it’s going to be used AJAX (Asynchronous JavaScript and XML), so to optimize the performances of the web site, since we built a RESTful service the asynchronous communication is optimized. All the services, exposed by the BackEnd for the web interface, use the informations stored in a Database. As DBMS ,YAAT is going to relay on MySQL, since it’s simple, open-source and performant. A DBMS is needed because YAAT is a Web Application that could possibly be used by many users at the same time, so a DMBS will take care of the concurrency issues and no performance drawback.
